Detailed Description

This class is instanced by the TracerUpdateSenderPlugin and is run in a subthread.

The class handles the connection to DataHub and the other TRACER clients, it builds a TRACER Update Message given a certain payload, like bool, int, float, vector of floats (vec2, vec3, vec4, quat, colour), or string

◆ run

void AnimHostMessageSender::run ( )
slot

Main loop, executes all operations.

Gets triggered by the Qt Application UI thread, when the "Send animation" button of the TracerUpdateSender Plugin is clicked. The first time it is called, it opens a socket. Then, for every frame present in the animation, it creates a message given the data coming from the other plugins (animation, character and scene data), and sends the message over the 0MQ connection.

◆ setAnimationAndSceneData()

void AnimHostMessageSender::setAnimationAndSceneData ( std::shared_ptr< Animation > ad,
std::shared_ptr< CharacterObject > co,
std::shared_ptr< SceneNodeObjectSequence > snl )

Setting data necessary to create new TRACER Update Message representing a character pose.

The character data is needed to get the parameter-to-bone mapping (which parameter modifies which bone). The bone is represented as the index of a node in the scene description. The node has a name, which can be matched to the name of a bone in the animation data, and from this it is possible to extract the pose of the bone to be sent as a TRACER Parameter Update Message

Parameters
[in]adThe animation data: collection of Bones, which contains a sequence of poses of a specific skeletal bone
[in]coThe character data: collection of all the data related to a character in a TRACER Application (e.g. Unity/Unreal/Blender VPET)
[in]snlThe scene description: collection of nodes in the Scene listed sequentially (not hierarchically)
