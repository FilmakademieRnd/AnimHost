3703 /* \brief Set affinities for intra op threads
3704 *
3705 * Affinity string follows format:
3706 * logical_processor_id,logical_processor_id;logical_processor_id,logical_processor_id
3707 * Semicolon isolates configurations among threads, while comma split processors where ith thread expected to attach to.
3708 * e.g. 1,2,3;4,5
3709 * specifies affinities for two threads, with the 1st thread attach to the 1st, 2nd, and 3rd processor, and 2nd thread to the 4th and 5th.
3710 * To ease the configuration, an "interval" is also allowed:
3711 * e.g. 1-8;8-16;17-24
3712 * orders that the 1st thread runs on first eight processors, 2nd thread runs on next eight processors, and so forth.
3713 * Note:
3714 * 1. Once set, the number of thread affinities must equal to intra_op_num_threads - 1,
3715 * ort does not set affinity on the main thread which is started and managed by the calling app;
3716 * 2. For windows, ort will infer the group id from a logical processor id, for example, assuming there are two groups with each has 64 logical processors,
3717 * an id of 64 will be inferred as the last processor of the 1st group, while 65 will be interpreted as the 1st processor of the second group.
3718 * Hence 64-65 is an invalid configuration, because a windows thread cannot be attached to processors across group boundary.
